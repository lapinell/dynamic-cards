Version 2: Implement Masonry Layout

If I have time, I will implement a version of Masonry.js so that the cards "fall up" the page when there are large gaps. For now, on to the next exercise.

There are actually a lot of things that I'd like to implement for this little exercise, that I may use on my personal site. Examples are:

  • When you hover, the date, category and tags of the post display on a dark background that overlays the post.
  • An admin section so the delete button and the new content text area isn't asseccible to everyone.
  • A class for image-posts.
  • Definitely implement some MD language so that html and js can't be popped in randomly on my site, and then I can also use headers and paragraphs freely.
  • And more...

Also, I learned if I get the scripts and html going then the CSS is pretty easy to implement afterwards.

Thanks to Jeremy for letting me "borrow his eyes".

That is all.
